새로운 기능을 이식할 때 다음 단계를 따라 분석을 수행합니다.

### 1단계: 레거시 코드 탐색
- `transfer/` 폴더에서 대상 기능을 담당하는 파일(들)을 식별합니다.
- 예: "빙고 로직 이식 시 `transfer/Systems/BingoSystem.js` 확인"

### 2단계: 핵심 로직 추출
- 데이터 구조(Data Structures), 수학 공식(Formulas), 조건식(Conditional Logics)을 추출합니다.
- React 전용 코드(State, Effect 등)를 제외한 순수 로직만 추려냅니다.

### 3단계: 유니티 매핑 설계
- 추출한 로직을 아래 유니티 컴포넌트 중 어디에 배치할지 결정합니다.
  - 데이터: `ScriptableObject`
  - 상태: `UnitEntity`
  - 엔진: `BattleManager` / `GridManager`

### 4단계: 변환 계획 리포팅
- 사용자에게 다음과 같은 형식으로 보고합니다.
  - **레거시 파일**: [파일명]
  - **핵심 로직 요약**: 
  - **유니티 구현 방향**: (예: ScriptableObject로 변환하여 에디터에서 제어)
  - **특이사항**: (레거시의 어떤 부분이 유니티에서 변경되어야 하는지)

### 5단계: 구현 승인
- 유저의 승인을 받은 후 실제 C# 코드 작성을 시작합니다.
